# 百日营 · AI Agent 任务分发配置（通用模板）

> 适用：任意组号的百日营学习进度跟踪系统。
> 使用方式：组长复制本文件，搜索并替换所有 `{...}` 占位符为自己的信息即可。

---

## 快速上手（3 步）

### 第 1 步：替换占位符

在本文档中搜索以下标记，替换为实际值：

| 占位符 | 替换为 | 示例 |
|--------|--------|------|
| `{组号}` | 你的组号 | `3` |
| `{组长名}` | 你的姓名 | `钟智敏` |
| `{开营日期}` | 开营日期 | `2026年5月21日` |
| `{组员名单}` | 组员姓名（逗号分隔） | `张三、李四、王五、…` |
| `{组员人数}` | 组员数量 | `9` |

### 第 2 步：准备好 Excel 跟踪表

确保工作目录下有以下文件（可由 AI 助手按此模板自动生成）：

```
百日营{组号}组学习跟踪表.xlsx     ← 核心数据文件
组员档案/                          ← Word 档案目录
```

Excel 跟踪表应包含 6 个工作表（详见下方结构）。

### 第 3 步：把本文档发给 AI 助手

对 AI 助手说：

> "请熟读 agents.md，这是我的百日营项目配置。现在帮我录入组员作业 / 刷新统计 / 生成周报。"

## Agent 定义

本项目定义 **5 个 Agent**，分别负责数据录入、统计报表、档案生成、进度监控、操作日志。每个 Agent 独立运行，通过 Excel 和 Markdown 文件进行数据交换。

---

### Agent 1：数据录入 Agent `data-entry`

**职责**：从群聊消息中解析作业提交、打卡、出勤数据，录入 Excel 跟踪表。

**触发条件**：
- 用户提供群聊截图或作业文本
- 用户指令中包含"录入""登记""更新数据""有人交作业了"等关键词
- 每周一上课后需登记出勤

**输入格式**：
```
[日期] [姓名] [作业类型] [作业内容]
例：5/25 张三 本周作业 今天和孩子发生了……觉察到自己……
```

**处理逻辑**：
1. 解析输入 → 提取日期、姓名、类型、内容
2. 写入「作业总览」表：按日期分组，新增行插入对应日期组
3. 写入「作业收集」表：追加新行，填入所有字段
4. 若为"每日觉察"或"每日践行"→ 同步更新「每日打卡明细」表的 ✓/✗ 标记
5. 若为上课出勤 → 写入「上课出勤记录」表
6. 标记 `已同步到档案` 列为"待更新"

**输出**：
- 更新后的 Excel 文件
- 操作摘要："已录入 X 条记录：姓名A(类型1)、姓名B(类型2)"

**注意事项**：
- 内容严格原封不动录入，不做润色或缩写
- AI 反馈列留空，由组长后续补充
- 同名同日期同类型作业视为更新（覆盖旧内容）

---

### Agent 2：统计报表 Agent `stats-report`

**职责**：刷新综合统计数据，生成进度看板图。

**触发条件**：
- Agent 1（数据录入）完成后自动触发
- 用户指令中包含"刷新统计""更新数据""看板""报表"等关键词
- 每周一上课后

**处理逻辑**：
1. 读取 Excel 全部6个工作表
2. 遍历「作业收集」表，按姓名+类型统计提交次数
3. 计算各项指标：
   - 觉察率 = 实际觉察次数 / 应觉察次数
   - 作业率 = 实际本周作业次数 / 应作业次数
   - 践行率 = 实际践行次数 / 应践行次数
   - 出勤率 = 出勤次数 / 应出勤次数
4. 写入「综合统计」表的百分比列（直接数值，非公式）
5. 更新活跃成员、未启动成员、部分参与成员分类
6. 更新「每日打卡明细」表的 ✓/✗ 标记
7. 可选：生成 `进度看板.png`

**输出**：
- 更新后的 Excel（综合统计表已刷新）

---

### Agent 3：档案生成 Agent `archive-gen`

**职责**：根据 Excel「作业收集」表，为每位组员生成/更新百日营 Word 档案。

**触发条件**：
- Agent 1（数据录入）完成后自动触发（仅更新有变化成员的档案）
- 用户指令中包含"生成档案""更新档案""刷新文档"等关键词

**处理逻辑**：
1. 读取「作业收集」表中 `已同步到档案` 列为空或"待更新"的行
2. 按姓名分组，提取每人所有作业内容
3. 读取「综合统计」表获取每人统计数据
4. 为每位有变化的组员生成/更新 Word 档案
5. 输出到 `组员档案/{姓名}_百日营档案.docx`
6. 更新「作业收集」表 `已同步到档案` 列为"✓"

**档案模板结构**：
```
# {姓名} · 百日营学习档案
## 基本信息
- 角色：组长/组员
- 开营日期：{开营日期}

## 学习进度
| 指标 | 完成 | 总计 | 完成率 |
|------|------|------|--------|
| 每日觉察 | X | — | X% |
| 本周作业 | X | — | X% |
| 每日践行 | X | — | X% |
| 上课出勤 | X | — | X% |

## 作业记录
### {日期} · {作业类型}
{作业内容原文}

💡 AI反馈：{如有}
```

---

### Agent 4：进度监控 Agent `monitor`

**职责**：定期生成周报，标记异常情况，提醒组长跟进。

**触发条件**：
- 每周日晚上（可按需调整）
- 用户指令中包含"周报""进度报告""检查""谁没交"等关键词

**处理逻辑**：
1. 读取「综合统计」表获取全组数据
2. 识别异常情况：
   - **未启动成员**：0提交的组员
   - **缺课成员**：出勤率为0%的组员
   - **下滑成员**：上次活跃但本次未提交
   - **新提交**：本次有新增提交的成员
3. 读取「每日打卡明细」检查近期打卡连续性
4. 生成结构化周报：

**周报模板**：
```markdown
# 百日营第{组号}组 · 第X周进度报告
**统计周期**：X月X日 - X月X日

## 📊 全组概览
| 指标 | 数值 |
|------|------|
| 活跃成员（≥3次提交） | X人 |
| 未启动成员（0提交） | X人 |
| 部分参与 | X人 |
| 本周出勤率 | X/X（X%） |

## ⚠️ 需要关注
- **XXX**：缺课缺作业，0参与

## ✅ 本周亮点
- **XXX**：提交了X次，内容优质

## 📋 下周待办
1. 跟进未启动成员参与状态
2. 第X周上课出勤登记
3. 持续跟踪每日践行打卡
```

---

### Agent 5：操作日志 Agent `operation-log`

**职责**：每次 Excel / 档案修改后自动记录操作日志，防止日志臃肿。

**触发条件**：
- Agent 1-4 任意一个完成后自动触发

**处理逻辑**：
1. 读取本次操作的摘要信息
2. 追加到 `操作日志.md` 当天日期段落中
3. **每日合并**：同一天同类操作合并为一行
4. **每周归档**：每周日自动将超过 7 天的内容归档到 `backup/操作日志/`

**防臃肿机制**：
- **行级合并**：同一天、同一操作类型、同一目标的多次修改合并为一行
- **周期归档**：每周日自动归档旧周
- **当前日志上限**：始终保持 ≤ 7 天的记录量

## Agent 协作流程

```
用户输入（群聊截图/作业文本）
        │
        ▼
  ┌─────────────┐
  │ Agent 1     │  数据录入 Agent
  │ data-entry  │  解析并录入 Excel
  └──────┬──────┘
         │ 
         ▼
  ┌─────────────┐
  │ Agent 2     │  统计报表 Agent
  │ stats-report│  刷新统计
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │ Agent 3     │  档案生成 Agent
  │ archive-gen │  更新组员 Word 档案
  └──────┬──────┘
         │
         ▼
  ┌─────────────┐
  │ Agent 5     │  操作日志 Agent
  │ operation-  │  记录日志 + 日合并
  │ log         │  周日自动周归档
  └─────────────┘

  ┌─────────────┐
  │ Agent 4     │  进度监控 Agent (独立触发)
  │ monitor     │  每周日定时 / 按需
  └─────────────┘
```

## 关键约束

1. **Excel 可靠性**：所有数据操作统一用 Python/openpyxl 一次性完成，避免多次保存导致 XML 损坏。如需多次编辑，在内存中完成所有修改后一次性保存。
2. **内容保真**：组员作业内容严格原文录入，不做任何润色、缩写或改写。
3. **AI 反馈独立性**：AI 反馈始终存放在独立列，不与原文混合。
4. **百分比处理**：综合统计表百分比使用直接数值填充，不使用 Excel 公式（避免公式不显示）。
5. **日志留存**：所有 Excel 写操作自动记录到 `操作日志.md`；同日同类合并，每周归档。

## 常见问答

**Q: 我没有 Excel 跟踪表，怎么开始？**
A: 把本文档发给 AI 助手，说"请根据这个 agents.md 帮我创建跟踪表，我的组员是：张三、李四、王五……，开营日期是 X月X日"。

**Q: 群聊截图里的作业怎么录入？**
A: 直接把截图发（或复制作业文字）给 AI 助手，它会自动解析并录入 Excel。

**Q: 怎么分享给组员看进度？**
A: 对 AI 说"生成周报"或"查看XXX的进度"，它会生成可转发到群的消息。

**Q: 数据安全吗？**
A: 所有数据存储在本地 Excel 文件中，不上传任何云端。本文档不含 API 密钥、连接接口或隐私信息。
